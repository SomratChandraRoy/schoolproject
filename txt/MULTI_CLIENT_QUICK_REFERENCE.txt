================================================================================
        Multiple Clients থেকে Ollama ব্যবহার - Quick Reference
================================================================================

🎯 Goal: 1টি Ollama server, 4টি client (Local + 3 Servers)

================================================================================

📋 Setup Order:
---------------
1. Ollama Server (AWS EC2) - Main AI server
2. Local Computer - আপনার laptop/desktop
3. Server 1 (AWS EC2) - First remote server
4. Server 2 (AWS EC2) - Second remote server
5. Server 3 (AWS EC2) - Third remote server

================================================================================

🚀 Quick Setup Commands:
------------------------

OLLAMA SERVER (Main EC2):
-------------------------
ssh -i ollama-key.pem ubuntu@OLLAMA_IP

# Enhanced config
sudo nano /etc/systemd/system/ollama.service.d/override.conf
# Add: OLLAMA_NUM_PARALLEL=4

# Install Nginx
sudo apt install nginx -y
# Configure load balancing (see full guide)

# Restart services
sudo systemctl daemon-reload
sudo systemctl restart ollama nginx

LOCAL COMPUTER:
---------------
# Python
pip install requests
python ollama-client-local.py

# Node.js
npm install axios
node ollama-client-local.js

# React App
cd frontend/medhabangla
echo "REACT_APP_OLLAMA_URL=http://OLLAMA_IP" > .env
npm run dev

SERVER 1, 2, 3:
---------------
ssh -i server-key.pem ubuntu@SERVER_IP
./setup_ollama_client.sh
# Enter: Ollama IP, Server name
ollama-ask "Test question"

📊 Monitoring:
--------------

Active Connections:
sudo netstat -an | grep :11434 | wc -l

Nginx Logs:
sudo tail -f /var/log/nginx/ollama_access.log

Ollama Logs:
sudo journalctl -u ollama -f

System Resources:
htop

🔒 Security:
------------

IP Whitelist (Nginx):
geo $allowed_ip {
    default 0;
    YOUR_IP 1;
    SERVER1_IP 1;
    SERVER2_IP 1;
    SERVER3_IP 1;
}

API Key:
X-API-Key: your-secret-key

HTTPS:
sudo certbot --nginx -d your-domain.com

💰 Cost Saving:
---------------

Stop when not in use:
aws ec2 stop-instances --instance-ids i-xxxxx

Schedule shutdown:
crontab -e
0 23 * * * sudo systemctl stop ollama

Use Elastic IP:
EC2 → Elastic IPs → Allocate

🔧 Troubleshooting:
-------------------

Connection Failed:
✓ Check: sudo systemctl status ollama
✓ Check: sudo systemctl status nginx
✓ Check: Security Group port 11434, 80

Slow Response:
✓ Check: htop (CPU/Memory)
✓ Reduce: OLLAMA_NUM_PARALLEL
✓ Use smaller model: gemma2:2b

Rate Limit:
✓ Increase: Nginx rate limit
✓ Check: /var/log/nginx/ollama_error.log
